LLM Building Blocks: A Plain-English Glossary

The terms you keep running into when building with language models — defined without the hand-waving.

Every time you build something on top of a language model, you hit the same wall of jargon: logits, autoregressive, embeddings, RAG. Most explanations either skip the definition entirely or bury it in math. This glossary defines the terms the way you actually need them — precise enough to be useful, plain enough to stick.

Term Plain English
Token A chunk of text. Could be a word, part of a word, or a single character, depending on the tokenizer. Models work in tokens, not characters or words.
Tokenizer The thing that chops text into tokens and assigns each token a number (an ID). It also reverses the process — turns IDs back into text.
Vocabulary The full list of tokens the tokenizer knows. GPT-2's is ~50,000 tokens.
Logits The raw scores the model outputs for every token in the vocabulary. A vector of ~50,000 numbers per generation step.
Softmax A math function that turns the score vector (logits) into probabilities that add up to 1. So "logits = scores, softmax(logits) = probabilities."
Decoding strategy The rule you use to pick which token to emit from the probability list — pick the highest one (greedy), sample with randomness (top-k/top-p), etc.
Autoregressive Fancy word for "generates one token at a time, feeding each new token back in to produce the next."
Embedding A way of representing a token (or a position) as a vector of numbers the model can do math on. The model can't multiply the word "cat" — it can multiply the vector that represents "cat."
Tensor A multi-dimensional array of numbers. In PyTorch, it's the basic data container — like a NumPy array but designed to run on GPUs and track gradients.
Model weights / parameters The billions of numbers learned during training that make the model "know stuff." When you load GPT-2 you're loading its weights.
Training Showing the model lots of text and adjusting its weights so its next-word predictions get better. You will not be doing this for the summarizer — GPT-2 is already trained. You will do it briefly in Phase 1.5 on a toy model, for understanding.
Inference Using a trained model to make predictions. That's what your summarizer does.
RAG Retrieval-Augmented Generation. Fancy term for "fetch some relevant external text and stick it into the prompt so the model has the facts it needs." Your project's "retrieval" is just fetching the URL the user gives you.
